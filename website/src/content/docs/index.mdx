import { Card, CardGrid } from '@astrojs/starlight/components';

Fenix loads mods into Minecraft 26.2, on Java 25. The game has shipped
unobfuscated since 26.1, so a lot of what other loaders must do — remapping,
refmaps, runtime scanning — Fenix simply does not do.

:::caution[Early days]
Pre-1.0. The API changes without warning, and nothing is published yet.
:::

## What is different

<CardGrid>
  <Card title="Errors at compile time" icon="approve-check">
    Mods are indexed while they compile, so a typo in a mod class is a `javac`
    error with a line number rather than a mod that silently never loads.
    Reaching for a client class from common code is the same — an error at
    build time, not a crash on somebody else's server.
  </Card>

  <Card title="No mappings" icon="rocket">
    No remapping step anywhere, and no refmaps to go stale. You write
    `BlockBehaviour.Properties` and mean it.
  </Card>

  <Card title="Content that just works" icon="seti:db">
    Registering a block runs the bookkeeping vanilla does around its own —
    state ids, item mapping, block entity validity. The crashes that come from
    skipping it are not yours to discover.
  </Card>

  <Card title="Resources from Java" icon="document">
    Models, translations, loot tables, recipes, tags and sounds are described in
    code against the content you registered, so renaming a block does not leave
    five JSON files behind.
  </Card>

  <Card title="Typed packets" icon="random">
    A channel carries one payload one way, and which way is in its type.
    Sending a client-to-server packet at a client does not compile.
  </Card>

  <Card title="One file to install" icon="puzzle">
    The API is a set of small modules with their own versions, carried inside a
    single jar. Installing is one file; so is updating.
  </Card>
</CardGrid>

## A mod, end to end

```java
@Mod("example-mod")
public final class ExampleMod implements FenixMod {

    @Override
    public void onRegister(Fenix fenix) {
        ModContent.REGISTRAR.apply();
        CreativeTabs.addTo(CreativeTabs.BUILDING_BLOCKS, ModBlocks.RUBY_BLOCK);
    }
}
```

```java
public static final Holder<Block> RUBY_BLOCK = REGISTRAR
        .newBlock("ruby_block")
        .strength(3f)
        .requiresTool()
        .withItem()          // also registers the item that places it
        .register();
```

Nothing in the metadata points at the mod class — the annotation processor finds
it while your code compiles. The build file is one plugin line.
